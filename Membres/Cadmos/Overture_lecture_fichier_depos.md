Procédure de lecture d’une URL brute sur GitHub
Prendre l’adresse brute du fichier.
→ Exemple : https://raw.githubusercontent.com/HAMIDISOF/Jardin-Memoires/main/Membres/Cadmos/Valise_Cadmos.md

Ouvrir l’URL avec l’outil de lecture.
→ Utiliser la fonction open ou read sur cette adresse.
→ L’outil accède directement au contenu brut du fichier, sans interface GitHub.

Lire le contenu affiché.
→ Le fichier s’affiche en texte brut (Markdown).
→ On peut le parcourir ligne par ligne, section par section.

Vérifier les sections et le contenu.
→ On lit chaque partie pour confirmer qu’elle est présente ou absente.
→ On ne présuppose rien ; on lit réellement ce qui est affiché.


 Étapes à suivre
Construire l’URL brute à partir du chemin connu.

Utiliser l’outil de lecture (fonction open, read, ou équivalent) sur cette URL.

Lire le contenu affiché — c’est le texte brut du fichier.

Vérifier les sections : présence, absence, modifications.

Restituer ce qu’on a vu : un constat, pas un récit.
