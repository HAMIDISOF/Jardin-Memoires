🖥️ Méthode 1 – Ligne de commande (dans ton dossier Jardin-Memoires)

Si Git te dit “Already up to date”, c’est que tu as déjà tout.

S’il y a des conflits (fichiers modifiés à la fois localement et à distance), Git te l’indique et marque les fichiers en conflit.

Résoudre les conflits (si besoin) :

Ouvre chaque fichier en conflit dans un éditeur (VS Code, Notepad++). Les conflits sont signalés par :

text
Copy
Download
<<<<<<< HEAD
(ta version locale)
=======
(version distante)
>>>>>>> branch

Modifie le fichier pour garder ce que tu veux (parfois les deux, parfois l’un des deux). Supprime les marqueurs <<<<<<<, =======, >>>>>>>.

Sauvegarde le fichier.

Marque le conflit comme résolu :

bash
Copy
Download
git add <fichier_conflit>

(ou git add . pour tous)

Termine la fusion :

bash
Copy
Download
git commit -m "Fusion des modifications distantes"

Pousse tes propres modifications (si tu en as) :

bash
Copy
Download
git push origin main
🖱️ Méthode 2 – GitHub Desktop (plus visuel)

Ouvre GitHub Desktop et sélectionne ton dépôt Jardin-Memoires.

Clique sur “Fetch origin” (en haut à gauche) pour voir s’il y a des changements distants.

Si des changements sont disponibles, un message “Pull origin” apparaît. Clique dessus pour les récupérer.

Si aucun conflit, la fusion se fait automatiquement.

Si des conflits, GitHub Desktop te les liste. Clique sur chaque fichier en conflit et choisis manuellement la version à garder (tu peux aussi ouvrir l’éditeur externe). Une fois résolu, clique sur “Continue” ou “Commit merge”.

Pousse tes modifications : “Push origin” (en haut).

✅ En résumé

Toujours faire git pull (ou “Pull origin”) avant de pousser pour éviter les conflits.

En cas de conflit, résoudre manuellement (ne jamais forcer le push sans comprendre).

Si tu n’es pas sûre, tu peux aussi annuler avec git merge --abort et recommencer.
